# Vector search & embeddings.
# chromadb is pinned to a narrow range because:
#   - 1.5.x has a Rust-based HNSW reader that can SIGSEGV on corrupt
#     segments rather than recover (see chroma#7069, #6975).
#   - We tune hnsw:sync_threshold=200 and rely on the single-writer
#     warm search-server (default-scripts/search-server.py) to avoid
#     concurrent PersistentClient opens, which the maintainers state
#     is not safe (chroma#3792).
# Different machines have shipped with different versions historically
# (laptop 1.4.x, Jetson 1.5.x). Keep them in lockstep going forward.
chromadb>=1.5.0,<1.6.0
sentence-transformers>=3.0.0
